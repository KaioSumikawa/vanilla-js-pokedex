# Vanilla JS Pokédex 

> Projeto de portfólio desenvolvido com **JavaScript puro (Vanilla JS)**, HTML e CSS, utilizando a **PokeAPI** para consumo de dados dos primeiros 151 Pokémon.


## Visão Geral

A **Pokédex Web Application** é uma aplicação frontend que simula uma Pokédex interativa, exibindo informações detalhadas dos Pokémon da primeira geração.

O projeto foi desenvolvido com foco em:

* Consumo de API REST (PokeAPI)
* Manipulação do DOM
* Requisições assíncronas (fetch / async-await)
* Organização de código frontend sem frameworks

## Objetivo do Projeto

Além do aspecto técnico, este projeto também foi pensado como exercício de:

* Estruturação de aplicações web simples
* Boas práticas de organização de código frontend
* Simulação de entrega de software em contexto profissional
* Introdução a conceitos de **governança de TI**, como:

  * Reprodutibilidade do código
  * Clareza de documentação
  * Manutenibilidade
  * Padronização de entrega

## Tecnologias Utilizadas

* HTML5
* CSS3
* JavaScript (Vanilla)
* PokeAPI ([https://pokeapi.co/](https://pokeapi.co/))

## Funcionalidades

* Listagem dos 151 primeiros Pokémon
* Busca/visualização em grid
* Exibição de:

  * Nome
  * Número na Pokédex
  * Imagem
  * Tipos
* Carregamento dinâmico via API

## Conceitos Aplicados

* Manipulação do DOM
* Consumo de API REST
* Promises e Async/Await
* Renderização dinâmica de componentes
* Organização de código frontend modular

## Relação com Governança em TI

Apesar de ser um projeto simples, ele pode ser relacionado a princípios de governança de TI, como:

* **Padronização:** estrutura clara de frontend sem frameworks
* **Transparência:** código aberto e documentado
* **Qualidade de entrega:** foco em legibilidade e manutenção
* **Reutilização:** lógica baseada em API pública (PokeAPI)
